O que é um repositório no Git e qual a sua importância?
R: Um repositório Git (ou repositório) é um local de armazenamento para um projeto, que contém todos os arquivos do projeto, a estrutura de pastas, e o histórico completo das alterações ao longo do tempo (os commits). Sua importância reside em permitir o versionamento do código, o rastreamento de mudanças, a colaboração entre desenvolvedores (mesmo remotamente) e a capacidade de reverter a um estado anterior do projeto, garantindo a segurança e organização do trabalho

Qual a função do Git Bash?
R:O Git Bash serve para fornecer um ambiente de linha de comando no Windows que emula os sistemas operacionais Linux e macOS, permitindo que os utilizadores utilizem comandos Git para gerir o versionamento de projetos e scripts .sh que não funcionariam nativamente no Windows.

O que faz o comando git add e qual a diferença entre git add <arquivo> e git add .?
R: Git add: adiciona seu diretório de trabalho à área de preparação, indicando o git quais arquivos ou mudanças devem ser incluídos no próximo commit.
 git add <arquivo>: adiciona as alterações de um arquivo específico ao índice ou área de staging, preparando-o para ser incluído no próximo commit enquanto o Git add . Adiciona todas as alterações  no diretório atual



Por que utilizamos o ponto (.) no comando git add .?

R: é utilizado pra dizer  oque o git deve adicionar a partido "."
